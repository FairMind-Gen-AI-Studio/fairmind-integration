fairmind-code-review Skill

Validation: Three-Layer Systematic Verification

LAYER 1: Plan Verification

Compare plan vs journal to detect scope issues

  • Get plan from get_task or work package
  • Read agent journal
  • Check: All planned items done?
  • Check: Any extra unplanned items? (scope creep)

LAYER 2: Requirements Verification

Use fairmind-context to verify all requirements met

  • Gather acceptance criteria, functional/technical requirements
  • Build checklist from requirements
  • Review code against checklist
  • Check test coverage expectations

LAYER 3: Cross-Repo Integration

Use Code tools to verify integrations

  • Identify API endpoints being called
  • Use Code_search to find API contracts
  • Verify request/response format matches
  • Use Code_find_usages to check for breaking changes

Review Output

Structured Report with:

• Executive summary

• Plan compliance results

• Requirements compliance checklist

• Integration verification results

• Critical issues / Warnings / Recommendations

APPROVED: All checks passed

CHANGES REQUESTED: Issues to fix

BLOCKED: Architectural review needed

Key Difference

Generic Review: Subjective quality checks

Fairmind Review: Objective 3-layer verification with full traceability (plan → journal → code)

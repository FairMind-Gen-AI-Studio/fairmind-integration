fairmind-tdd Skill

Development: Test-Driven with Acceptance Criteria Alignment

SETUP: Gather Context

Use fairmind-context → Get plan, acceptance criteria, test expectations

RED: Write Failing Test

Test aligned with Fairmind acceptance criteria (not invented). Run to verify failure.

GREEN: Minimal Implementation

Write minimal code following get_task plan (not free-form). Make test pass.

REFACTOR: Clean Code

Improve quality while keeping tests green. Remove duplication.

DOCUMENTATION: Update Journal

Document: plan item → test → code. Provides traceability.
